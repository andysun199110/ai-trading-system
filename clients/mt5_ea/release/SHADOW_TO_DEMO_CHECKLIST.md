# Shadow → Demo → Live 切换检查清单

## 📋 阶段说明

Gold AI Trader EA 采用**三阶段渐进式部署**策略，确保系统稳定性和资金安全：

| 阶段 | ShadowMode | AllowTrading | 目的 | 风险 |
|------|------------|--------------|------|------|
| **Shadow（影子）** | `true` | `false` | 验证系统连接和信号接收 | 无 |
| **Demo（模拟）** | `true` | `false` | 在模拟账户验证策略表现 | 无 |
| **Live（实盘）** | `false` | `true` | 真实资金交易 | 有 |

**⚠️ 重要原则：**
- 每个阶段必须完成所有检查项才能进入下一阶段
- 建议每个阶段至少运行 **7-14 天** 观察表现
- 不要跳过任何阶段！

---

## 🌑 第一阶段：Shadow Mode（影子模式）

### 目标
- 验证 EA 能正常连接 API 服务器
- 验证身份认证通过
- 验证能正确接收和解析信号
- 观察信号质量和频率
- **不执行任何实际交易**

### 配置要求
```
Inp_ShadowMode       = true
Inp_AllowTrading     = false
Inp_ApiBaseUrl       = https://your-api-domain.com/api/v1
Inp_LicenseKey       = <your-license-key>
```

### ✅ 检查清单

#### 1. 基础连接验证
- [ ] EA 成功添加到 XAUUSD 图表
- [ ] 状态面板显示 "✅ Connected"
- [ ] 专家日志显示 "Initial authentication successful"
- [ ] 无 HTTP 错误（0, 401, 403, 500 等）

#### 2. 心跳验证
- [ ] 日志中定期出现 "💓 Heartbeat sent successfully"
- [ ] 心跳间隔与 `Inp_PollIntervalSec` 一致（默认 30 秒）
- [ ] 连续 24 小时无心跳失败

#### 3. 信号接收验证
- [ ] 日志中出现 "📡 Poll: No new signals"（无信号时）
- [ ] 当服务器有信号时，日志显示 "📊 New signal: xxx"
- [ ] 信号解析正确（action、lot_size、symbol 等字段）

#### 4. 影子模式执行验证
- [ ] 收到信号时，日志显示 "🔒 SHADOW MODE: Would execute XXX (NOT EXECUTED)"
- [ ] **确认没有实际订单产生**（检查 MT5 交易历史）
- [ ] 信号记录到 `GoldAI_processed_signals.csv` 文件

#### 5. 稳定性验证
- [ ] EA 连续运行 24 小时无崩溃
- [ ] MT5 重启后 EA 自动恢复运行
- [ ] 网络连接中断后能自动重连

#### 6. UI 面板验证
- [ ] 状态面板正常显示
- [ ] 连接状态实时更新
- [ ] 最后轮询时间正常更新

建议记录以下数据（至少 7 天）：

| 日期 | 信号数量 | 认证成功率 | 心跳失败次数 | 备注 |
|------|----------|------------|--------------|------|
| Day 1 | | | | |
| Day 2 | | | | |
| ... | | | | |

### 🎯 Shadow 阶段完成标准

**必须全部满足才能进入 Demo 阶段：**

- [ ] 连续 7 天稳定运行，无崩溃
- [ ] 认证成功率 > 99%
- [ ] 心跳失败次数 < 5 次/天
- [ ] 信号接收正常，解析无误
- [ ] 确认影子模式生效（无实际交易）
- [ ] 已记录并分析所有接收到的信号

## 🌓 第二阶段：Demo Mode（模拟盘模式）

### 目标
- 在模拟账户上验证策略实际表现
- 验证信号执行逻辑（虽然仍在 Shadow 模式）
- 观察策略盈亏表现
- 调整风险参数
- **仍不执行真实交易**

### 配置要求
```
Inp_ShadowMode       = true        ⚠️ 仍建议保持 true
Inp_AllowTrading     = false       ⚠️ 必须为 false
Inp_AccountServer    = <demo-server>
Inp_MaxLotCap        = 0.1         根据模拟账户规模调整
```

### ✅ 检查清单

#### 1. Demo 账户准备
- [ ] 已开设模拟账户（推荐 TradeMaxGlobal-Demo 或类似）
- [ ] 模拟账户资金规模接近计划实盘规模
- [ ] 记录模拟账户初始余额

#### 2. 参数优化
- [ ] 根据 Shadow 阶段观察调整 `Inp_MaxLotCap`
- [ ] 确认 `Inp_StopLossPoints` 和 `Inp_TakeProfitPoints` 合理
- [ ] 验证 `Inp_PollIntervalSec` 适合当前策略

#### 3. 信号执行验证
- [ ] 收到 BUY 信号时，日志显示正确的执行计划
- [ ] 收到 SELL 信号时，日志显示正确的执行计划
- [ ] 收到 CLOSE 信号时，日志显示正确的平仓计划
- [ ] 信号执行逻辑与预期一致

#### 4. 策略表现跟踪
- [ ] 每日记录信号数量和方向
- [ ] 手动计算如果执行交易的理论盈亏
- [ ] 对比 EA 计算的预期盈亏
- [ ] 观察最大回撤是否在可接受范围

#### 5. 极端情况测试
- [ ] 观察网络中断时的处理（应暂停交易）
- [ ] 观察服务器不可用时的处理（应使用 fallback 或暂停）
- [ ] 观察异常信号的处理（如超大手数，应被 MaxLotCap 限制）

建议记录以下数据（至少 14 天）：

| 日期 | 信号数 | 理论盈亏 | 最大回撤 | 胜率 | 备注 |
|------|--------|----------|----------|------|------|
| Day 1 | | | | | |
| Day 2 | | | | | |
| ... | | | | | |

### 🎯 Demo 阶段完成标准

**必须全部满足才能考虑进入 Live 阶段：**

- [ ] 连续 14 天稳定运行
- [ ] 理论盈亏曲线符合预期
- [ ] 最大回撤 < 计划承受范围（如 <10%）
- [ ] 胜率 > 50%（或符合策略预期）
- [ ] 风险参数（止损、止盈、手数）已优化
- [ ] 已验证极端情况处理逻辑
- [ ] **已充分理解策略风险和收益特征**

## ☀️ 第三阶段：Live Mode（实盘模式）⚠️

### ⚠️ 重要警告

**进入此阶段前，请再次确认：**

- [ ] 已完成 Shadow 阶段所有检查项
- [ ] 已完成 Demo 阶段所有检查项
- [ ] 已充分理解策略风险
- [ ] 投入资金为可承受损失范围
- [ ] 已设置合适的风险参数
- [ ] 已告知相关人员（如适用）

**实盘交易有风险，入市需谨慎！**

### 配置要求
```
Inp_ShadowMode       = false       ⚠️ 仅在此阶段设为 false
Inp_AllowTrading     = true        ⚠️ 仅在完全验证后设为 true
Inp_AccountServer    = <live-server>
Inp_MaxLotCap        = 0.01        ⚠️ 初始建议最小手数
Inp_RequireServerRiskParams = true
Inp_UseFallbackRiskParams   = false
```

### ✅ 检查清单

#### 1. 实盘账户准备
- [ ] 已开设真实交易账户
- [ ] 账户已完成实名认证和入金
- [ ] 记录账户初始余额
- [ ] 确认账户支持 XAUUSD 交易

#### 2. 风险控制配置
- [ ] `Inp_MaxLotCap` 设置为保守值（建议 0.01 起始）
- [ ] `Inp_StopLossPoints` 和 `Inp_TakeProfitPoints` 已确认
- [ ] 账户总风险敞口在可承受范围内
- [ ] 已设置账户级别止损（如 MT5 的 Stop Out 级别）

#### 3. 参数切换
- [ ] `Inp_ShadowMode` 从 `true` 改为 `false`
- [ ] `Inp_AllowTrading` 从 `false` 改为 `true`
- [ ] 确认 API URL 指向生产环境
- [ ] 确认 License Key 为生产环境密钥

#### 4. 切换验证
- [ ] 修改参数后 EA 重新初始化
- [ ] 日志显示 "⚠️ LIVE (Real Trading)" 模式
- [ ] 状态面板模式显示正确
- [ ] **先观察 1-2 个信号，确认执行正常**

#### 5. 首笔交易验证
- [ ] 收到首笔交易信号
- [ ] 日志显示 "🚀 Executing XXX"
- [ ] MT5 交易历史中出现对应订单
- [ ] 订单手数、止损、止盈与预期一致
- [ ] 订单执行价格无明显滑点

#### 6. 持续监控
- [ ] 每日检查交易执行情况
- [ ] 每周复盘交易表现
- [ ] 每月评估策略有效性
- [ ] 根据实际表现调整风险参数

### 📊 Live 阶段监控指标

| 日期 | 交易数 | 盈亏 | 累计盈亏 | 最大回撤 | 备注 |
|------|--------|------|----------|----------|------|
| Day 1 | | | | | |
| Day 2 | | | | | |
| ... | | | | | |

### 🎯 Live 阶段最佳实践

1. **从小开始：** 初始使用最小手数（0.01），逐步增加
2. **密切监控：** 前两周每天检查，之后可适当减少频率
3. **定期复盘：** 每周复盘交易表现，每月评估策略
4. **风险优先：** 任何时候都以风险控制为第一优先级
5. **及时止损：** 如表现持续不佳，及时暂停并分析原因

## 🔄 阶段回退机制

**如遇到以下情况，应立即回退到前一阶段：**

### Live → Demo 回退
- [ ] 连续亏损超过预设阈值（如 >10%）
- [ ] 发现系统 bug 或异常行为
- [ ] 策略表现与 Demo 阶段差异过大
- [ ] 市场环境发生重大变化

**回退操作：**
```
Inp_ShadowMode   = true
Inp_AllowTrading = false
```

### Demo → Shadow 回退
- [ ] 发现信号解析错误
- [ ] 连接稳定性问题
- [ ] 需要修改核心参数
- [ ] 策略逻辑需要调整

**回退操作：** 保持 Shadow 配置，修复问题后重新验证
